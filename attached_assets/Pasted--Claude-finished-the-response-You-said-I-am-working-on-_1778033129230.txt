The three new bottlenecks
BN-05 (critical) — Prewarm silently failed. The prewarm log shows prewarmMs=11.1ms for 3× rpm.render(0). A single WebGPU PSO compile takes ~80–400ms minimum. 11.1ms total is a complete no-op — the renders returned instantly. The most likely cause is the WebGPU device-loss/recovery cycle visible earlier in the same log: [RenderPipelineManager] Pipeline init error: Cannot read properties of undefined (reading 'usedTimes') — the pipeline was in phase: error state, and render(0) silently no-ops in that state. Fix: Three guards: (1) abort prewarm if rpm.status.phase !== 'active', (2) if prewarm completes in < 30ms, do NOT set _shadersPrewarmed = true — treat it as failed and retry on next execute, (3) call __resetCwPrewarm() inside the existing webglcontextlost and WebGPU device-loss handlers so context recovery always forces a valid re-prewarm.
