## What's actually happening

All the fixes in your code are architecturally correct — but they're all calling `renderer.compile()` on **the wrong renderer**. Here's the chain:

- `window.bimWorld.renderer` → OBC's `PostproductionRenderer` (WebGL)
- `window.pryzmRenderer` → the actual `WebGPURenderer` that drives all production rendering after Phase 5 activates (set at `initScene.ts` line 1125)

In Phase 5, OBC is locked to `MANUAL` mode and never renders again. Every real frame goes through `pryzmRenderer`. So `renderer.compile()` in `_prewarmCurtainWallShaders()` and in `endBatchRenderSuppress()` both compile PSOs into the WebGL context — and those PSOs are completely discarded when `pryzmRenderer.render()` fires in `_obcCallback` (actually the Pascal callback) and does a cold WebGPU compile. That's your 10,923ms LONGTASK.
