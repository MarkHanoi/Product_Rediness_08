### Fix 1 — `CreateCurtainWallsOnAllSlabsCommand.ts`: Replace the prewarm with `renderer.compile()`

The current `_prewarmCurtainWallShaders()` renders via `window.pryzmRenderer` — but OBC's `PostproductionRenderer` is a different instance with a different WebGPU pipeline. The PSOs it compiles are **never reused** by OBC. The fix is to call `renderer.compile(scene, camera)` on the **actual OBC renderer** before lifting batch suppression, not before.

**Exact change — replace the entire `_prewarmCurtainWallShaders()` method body** (lines 500–610 of your file):

```typescript
private _prewarmCurtainWallShaders(): void {
    // §PERF-PREWARM-V2 (2026-05-05): The previous approach called window.pryzmRenderer.render()
    // but OBC uses PostproductionRenderer — a separate WebGPU device context. PSOs compiled by
    // pryzmRenderer are NOT in OBC's pipeline cache. This is why the 10,923ms LONGTASK fires on
    // first real render despite prewarm.
    //
    // New approach: call renderer.compile(scene, camera) on the OBC renderer DIRECTLY.
    // renderer.compile() does a full PSO compilation pass in the correct pipeline context
    // without actually drawing a frame — it is the canonical Three.js API for pre-compilation.
    // This eliminates the first-render LONGTASK entirely.
    //
    // We add three probe meshes (matching the exact material configs used in build()) so
    // renderer.compile() picks up InstancedMesh + DoubleSide glass + fallback Mesh variants.
    // All probes are invisible (scale 0) and removed immediately after compile().
    try {
        const scene    = window.bimWorld?.scene?.three as THREE.Scene | undefined;
        const camera   = window.bimWorld?.camera?.three;
        // KEY: use the OBC renderer, not window.pryzmRenderer
        const renderer = window.bimWorld?.renderer?.three;
        if (!scene || !camera || !renderer) {
            console.warn('[CreateCurtainWallsOnAllSlabsCommand] PERF-PREWARM-V2: OBC renderer not ready — skipping compile().');
            return;
        }

        const probeGeom = new THREE.BoxGeometry(0.001, 0.001, 0.001);

        const mullionMat = new THREE.MeshStandardMaterial({ color: '#333333', metalness: 0.1, roughness: 0.2 });
        const mullionIM = new THREE.InstancedMesh(probeGeom, mullionMat, 1);
        mullionIM.scale.setScalar(0); // invisible
        mullionIM.castShadow = false;
        mullionIM.receiveShadow = false;
        mullionIM.setMatrixAt(0, new THREE.Matrix4());
        mullionIM.instanceMatrix.needsUpdate = true;

        const glassMat = new THREE.MeshStandardMaterial({ color: 0x88ccff, transparent: true, opacity: 0.4, metalness: 0.1, roughness: 0.1, side: THREE.DoubleSide });
        const glassIM = new THREE.InstancedMesh(probeGeom, glassMat, 1);
        glassIM.scale.setScalar(0);
        glassIM.castShadow = false;
        glassIM.receiveShadow = false;
        glassIM.setMatrixAt(0, new THREE.Matrix4());
        glassIM.instanceMatrix.needsUpdate = true;

        const fallbackMat = new THREE.MeshStandardMaterial({ color: 0x88ccff, transparent: true, opacity: 0.4, metalness: 0.1, roughness: 0.1 });
        const fallbackMesh = new THREE.Mesh(probeGeom, fallbackMat);
        fallbackMesh.scale.setScalar(0);
        fallbackMesh.castShadow = false;
        fallbackMesh.receiveShadow = false;

        scene.add(mullionIM);
        scene.add(glassIM);
        scene.add(fallbackMesh);
        try {
            // renderer.compile() compiles ALL PSOs for everything currently in the scene
            // (existing geometry + our 3 probes) in the correct OBC pipeline context.
            // This is synchronous and takes ~200–400ms — same cost as before but now
            // the PSOs it produces are actually used by OBC on first render.
            renderer.compile(scene, camera);
        } finally {
            scene.remove(mullionIM);
            scene.remove(glassIM);
            scene.remove(fallbackMesh);
            probeGeom.dispose();
            mullionMat.dispose();
            glassMat.dispose();
            fallbackMat.dispose();
        }
        console.log('[CreateCurtainWallsOnAllSlabsCommand] PERF-PREWARM-V2: renderer.compile() complete (OBC context — PSOs will be cache-hit on first render).');
    } catch (e) {
        console.warn('[CreateCurtainWallsOnAllSlabsCommand] PERF-PREWARM-V2: compile() failed (non-fatal):', e);
    }
}
```

**Why `window.bimWorld.renderer.three`**: `bimWorld` is the OBC `SimpleWorld` — `.renderer.three` gives you the underlying `THREE.WebGPURenderer` that OBC uses. This is the same context that fires on the first real render. Verify the path exists with `window.bimWorld?.renderer?.three` — if your OBC version exposes it differently (e.g. `postproductionRenderer.renderer`) update that one path only.
