**Fix 3 — Shadow reactivation: remove the 3000ms timeout entirely:**

The `timeout: 3000` on `requestIdleCallback` is backfiring. When the browser has been frozen for 4+ seconds with LONGTASKs, it fires the idle callback as soon as any idle moment exists — which happens to be right when the next LONGTASK is starting. This creates the 16.6 second pile-up.

Replace with a much longer timeout and a guard that checks if LONGTASKs are still active:

```typescript
// In CurtainWallBuilder._reactivateShadows()
private _reactivateShadows(): void {
    // Don't fire shadow reactivation until frame rate recovers
    // Check FPS first — if still under stress, defer further
    const waitForStableFrames = (callback: () => void, minFps = 20) => {
        const check = () => {
            const fps = window.__currentFps ?? 0;  // expose from FPS monitor
            if (fps >= minFps) {
                requestIdleCallback(callback, { timeout: 10000 });
            } else {
                setTimeout(() => check(), 500);  // retry in 500ms
            }
        };
        setTimeout(() => check(), 2000);  // initial 2s cooldown
    };
    
    waitForStableFrames(() => {
        this._drainShadowSlice();
    });
}
```

This delays shadow reactivation until FPS recovers to at least 20fps, preventing it from piling onto an already-stressed main thread.
