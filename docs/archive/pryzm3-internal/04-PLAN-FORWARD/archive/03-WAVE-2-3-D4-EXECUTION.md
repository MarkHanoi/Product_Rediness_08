## §5 — When a slice's importer cluster turns out larger than estimated

This is the most likely Wave 2-3 risk (R1 of `13-RISK-REGISTER.md`).

### Decision tree

```
Re-estimate importer cluster at slice kickoff (Day 1 of slice).

Real cluster size vs estimate:
  ≤ 1.2× estimate  →  proceed normally
  1.2× – 1.5×      →  proceed with +1 day; reviewer aware
  ≥ 1.5×           →  SPLIT the slice
```

### Split mechanic

If D.4.1's 28-file cluster turns out to be 45 files at kickoff:

1. Architecture-lead splits: D.4.1a (first 22 files, half the LOC) + D.4.1b (next 23 files, half the LOC).
2. D.4.1a takes the original Wave 2 day 1–5 slot.
3. D.4.1b becomes the **first** PR of Wave 3 (Mon week-5), pushing D.4.3 to Tuesday and tightening D.4.4–D.4.5 by 1 day each.
4. Net schedule impact: 0 days (Wave 3 is a 5-PR week if a split happens; otherwise 3-PR week).

### What must NOT happen

- A slice is **not** allowed to balloon to 2× LOC budget mid-PR. Either it fits the budget or it splits. The architecture-lead enforces this at review.
- A split is **not** allowed to fold two slices into one mega-PR. The whole point of D.4 being 5 PRs is the per-slice rollback safety.

## §6 — Rollback procedures

### Per-slice rollback

```bash
# Revert the slice's merge commit
git revert -m 1 <slice-merge-sha>

# Re-baseline the cast/raf/loc tripwires (the post-revert state is the new baseline)
pnpm ga-gate --check loc-tripwire   # auto-ratchets
pnpm ga-gate --check cast-tripwire  # auto-ratchets

# Re-quarantine the previously-de-quarantined workflow if revert exposes it
git mv packages/<pkg>/__tests__/<spec>.spec.ts packages/<pkg>/__tests__/quarantined/

# Document the revert in ../03-CURRENT-STATE.md §10 weekly delta
```

### Wave-level rollback

Reverting an entire wave is **not** advised — it would reintroduce 2 weeks of debt. Instead:

1. Keep the merged slices as-is.
2. Roll forward with the next slice on a hotfix schedule.
3. The hotfix slice's PR description names the regression and the recovery plan.

### Wave 3 rollback exception

If D.4.5 (the shim) breaks the production startup path (e.g. `composeRuntime()` returns an incomplete runtime that some panel can't tolerate), the rollback is to D.4.4:
1. `git revert <D.4.5-sha>` — restores `EngineBootstrap.ts` to ~630 LOC.
2. Production runs from `EngineBootstrap` again (the slices D.4.1–D.4.4 are still merged; their work is preserved).
3. Wave 3 closes 1 sprint late; D.4.5 redo is the first PR of S81-WIRE.
