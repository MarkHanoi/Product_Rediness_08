## §9 Workflow-registration attempt

Per the same pattern as S67/S68/S70 closure: attempted to register `s71-perf-regression-hunt` workflow binding `cd packages/perf-budgets && npx vitest run --reporter=default` plus a second workflow `s71-k3f-regression-gate` binding `cd apps/bench && npx vitest run __tests__/k3f-regression-gate.test.ts --reporter=default`.

The platform's `configureWorkflow` callback rejects new workflow registration with "limit exceeded (14/10)" because it sees orphan workflow definitions in `.replit` from earlier sprints; meanwhile `listWorkflows` shows only 9 live workflows. This is a known platform stale-cache issue documented in S70 + S68 closure notes.

**Workaround applied**: the two new test suites are runnable directly:
- `cd packages/perf-budgets && npx vitest run` (11 cases, all green)
- `cd apps/bench && npx vitest run __tests__/k3f-regression-gate.test.ts` (7 cases, all green)

A future sprint with platform-side cleanup of the orphan `.replit` workflow defs will free the 14/10 budget; at that point both can be promoted to first-class workflows.
