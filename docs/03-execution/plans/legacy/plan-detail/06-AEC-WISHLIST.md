> The original `12-` roadmap addressed standards (ISO 19650, IFC4, COBie, IDS, ICDD, bSDD), AI design-partner, linked data, digital twin. The AEC Magazine BIM 2.0 wishlist also flags: stakeholder collaboration as the Figma-of-BIM wedge; hybrid local/cloud sovereignty (Graphisoft lesson); browser security as enterprise gate; **AI-automated AND CHECKED 2D drawing output** as the single highest-value AI feature; DfMA / digital fabrication; sustainability + LCA + embodied carbon; outcome-based pricing models; ConTech bridges across the design-construction divide. This supplement folds those into the roadmap as 8 new SPECs and renumbers everything downstream.

## §5 Resolution rule for conflicts with `12-` doc

The original `12-BIM-2-AND-3-POST-GA-ROADMAP.md` text retains its strategic narrative (§1 framing, §8 standards leadership, §9 the four bets). Where the original `12-` doc cites SPEC numbers, this supplement's renumbering wins. Where the original cites a sprint, this supplement's per-phase doc wins. Where this supplement adds a SPEC not in the original (33, 34, 35, 43, 46, 47, 53, 58), the SPEC file itself is the single source of truth.

Conflict order: **SPEC > ADR > 13-supplement > 12-roadmap > phase docs**.
