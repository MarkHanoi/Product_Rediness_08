# 0065 — Geodata analytical layers are a first-class pluggable-provider subsystem draped on Forma/Cesium (not hardcoded per-country, not nothing)

**Status**: PROPOSED
**Date**: 2026-06-09
**Deciders**: architecture team (founder-driven — reference: Hektar, a Nordic early-stage land-analysis tool, with a country-grouped Layers panel of analytical geodata draped over a 3D massing+terrain view, each layer a toggle + opacity slider; data credited to Lantmäteriet, SGU, Mapbox, Nimbo)

## Context

PRYZM already has a Forma-style 3D site view (`apps/editor/src/ui/geospatial/CesiumViewport.ts`, `FormaSiteAnalysisControls.ts`; SPEC-FORMA-SITE-VIEW), keyless basemaps (OSM streets / ESRI satellite / Google 3D Tiles), context buildings, and toggleable **climate** overlays — sun scrubber, soft shadow, wind rose + 3D wind streaks, comfort heat field (A.21.D24). It even already carries a `HEKTAR_PALETTE` 2D cartography style (`siteMap2DStyle.ts`) inspired by the same reference tool. What it does **not** have is the analytical-geodata dimension that makes an early-stage land tool useful: the **site constraints** a designer needs from day one — flood (100/200-year/calculated-max), landslide susceptibility, terrain slope/shading, soil types, ground coverage, property/detail plans, ancient monuments, drainage basins, population density, noise pollution, protected (Natura 2000) habitats — draped over the terrain + massing.

The founder shared the reference's UX explicitly: a **Layers panel** on the right, layers **grouped by country** (Denmark / Norway / Sweden) plus a cross-country top group (Europe), each layer a **toggle with an opacity slider**, layers **draping** over the 3D terrain + white building massing, with each data source **credited** (Lantmäteriet, SGU, Mapbox, Nimbo). The decision is **how PRYZM should provide analytical geodata layers**: hardcoded per-country, a pluggable-provider subsystem, or not at all. The founder explicitly asked for a **contract + strategy + spec + ADR**.

## Decision

**PRYZM provides analytical geodata as a FIRST-CLASS, PLUGGABLE-PROVIDER layer subsystem draped on the existing Forma/Cesium 3D site view — declarative layer descriptors, a single `GeodataProvider` interface, a country-grouped Layers panel (toggle + opacity), with mandatory per-source attribution and a C22 privacy tier for sensitive layers.**

- **Pluggable provider, not hardcoded country.** National registries plug in through one `GeodataProvider` interface; adding Norway/Denmark is registering an adapter, never editing core (C55 §1.3). The **reference adapter is Swedish Lantmäteriet + SGU**; a generic **OGC (WMS/WMTS/WFS) adapter** is the standards fallback.
- **Layers DRAPE, they are not BIM** (C55 §1.2) — read-only analytical context over the Cesium terrain + Forma massing; raster via Cesium `ImageryLayer`, vector via clamped-to-ground / draped geometry through the THREE owner (C04). Never committed to the model.
- **It reuses the existing site substrate** — the Site/parcel bbox (C19) in the C12 LTP-ENU/WGS84 frame, alongside the existing climate overlays and the C54 wind-CFD field as siblings on the same view.
- **Attribution is mandatory** (C55 §1.5, C23); **population/noise carry the C22 tier** (C55 §1.6); **opacity is render-only, no re-fetch**, tiles are lazy + cached (C55 §1.7).
- **Visualisation only for GL.1–GL.5**; the flood/landslide/slope-as-generator-keep-out tie-in is a reserved future seam feeding an EXISTING constraint input, not a new objective (C55 §1.8).

The normative invariants live in C55; the engineering design in SPEC-GEODATA-ANALYTICAL-LAYERS; the strategic framing in `docs/01-strategy/site-and-cognition-strategy.md §2.6`. This ADR records *why pluggable-provider draped layers* over the alternatives.

## Consequences

- **Positive:** early-stage land constraints (flood / landslide / slope / soil / noise / habitats / population) are visible from day one on the view designers already use; discipline-neutral (any typology — residential, commercial, public — benefits, not just apartments); builds on the existing Forma/Cesium/site substrate (no new viewer, no new coordinate frame); the pluggable model lets coverage grow per-country (and via the OGC fallback, per any standards-compliant registry) without touching core; attribution + privacy tier are baked in as contract invariants; a clean future seam to feed flood/landslide as generator keep-outs.
- **Negative / trade-offs:** geodata coverage is sparse and per-region → the panel must degrade gracefully layer-by-layer (C55 §1.4) and many regions show "no data"; national registries differ in CRS, format, licence, and rate limits → the provider adapters (esp. the Lantmäteriet/SGU reference) are real per-registry engineering; a new L1/L2 package + schemas + panel + adapters is non-trivial effort (the GL.1–GL.5 plan); raster/vector tiling adds a perf surface (mitigated by lazy-load + cache + opacity-without-re-fetch, C55 §1.7); per-country sovereignty (C49) and PII tier (C22) add gating cost for sensitive layers.
- **Provenance:** every active layer's source + dataset version + fetch timestamp + licence is recorded (C23) so an AI consumer (future §1.8 tie-in) inherits auditable provenance.

## Alternatives considered

- **Hardcode per-country layers directly in the Forma view.** Rejected: freezes the subsystem to whichever country ships first (Sweden), forces a core edit for every new registry, and entangles country-specific CRS/format/licence logic into L1/L2 — the opposite of the layered model. The pluggable-provider interface is the whole point.
- **Do nothing — keep only the climate overlays + basemaps.** Rejected: the basemaps show *imagery*, the climate overlays show *sun/wind/shadow* — neither shows the *land constraints* (flood/landslide/slope/soil/noise/habitats/population) that are the reason an early-stage land tool exists. Without them PRYZM's site view is a pretty massing study, not a land-analysis surface.
- **A single global geodata provider (e.g. one commercial aggregator).** Rejected for the core: it would couple PRYZM to one vendor's coverage, pricing, and licence, and still wouldn't reach the authoritative national registries (Lantmäteriet/SGU and equivalents) that early-stage land analysis needs. A commercial aggregator can still register *as a provider* under the same interface — but it cannot *be* the interface.
- **Commit geodata as BIM site elements (terrain mesh, flood polygons as model geometry).** Rejected: analytical context is not authored design; baking it into the model pollutes the `.pryzm` file, schedules, and IFC export with non-authored data and conflates "context for the design" with "the design." Layers drape; they are read-only (C55 §1.2).
