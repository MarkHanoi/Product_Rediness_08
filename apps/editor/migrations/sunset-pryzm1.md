## §5 Sunset banner + customer-comms cadence

Per SPEC-27 §3.2 + ADR-021, the 90-day sunset window opens on the day the
default flips (S61 D5). Until then, PRYZM 1 sessions see no change. After
the flip:

- **Day 0 (S61 D5)**: Sunset banner appears at the top of every PRYZM 1
  session: "PRYZM 1 will be removed on `<date>` (90 days). Visit `pryzm.app/sunset`
  for migration help." Dismissible per-session.
- **Day 30**: Banner upgrades to weekly modal on session start.
- **Day 60**: Banner becomes blocking; user must click "Continue with PRYZM 1"
  to dismiss, with a "Migrate now" CTA linking to `tools/pryzm1-sunset/`.
- **Day 90 (S70)**: PRYZM 1 boot path returns an HTTP 410 Gone response;
  `src/engine/` deleted; only `?pryzm2=1` (now default — flag is a no-op) works.

The countdown is calendar-time, not sprint-time, so the deletion sprint S70
must wait for the 90-day clock to run out even if S70 finishes early.
