Revit’s behavior here is a **very important concept in BIM modeling**: the **host–boundary constraint system**.
When you *“pick walls”* for a slab boundary, Revit is not just drawing lines — it is creating **associative geometric constraints between elements**.

Let’s break down **how Revit actually does it internally**.

---

# 1️⃣ Slabs Are Not Pure Geometry — They Are Constraint-Driven Elements

When you create a floor/slab in Autodesk Revit, the slab has:

### A **semantic element**

```
FloorElement
  id
  levelId
  typeId
  thickness
  sketchId
```

### A **sketch definition**

```
Sketch
  loops[]
     edges[]
```

Each **edge** in the sketch can be one of two types:

```
EdgeType
- FreeLine
- HostReference
```

---

# 2️⃣ What Happens When You “Pick Walls”

When you click **Pick Walls** in the floor sketch tool:

Revit does **NOT** store the wall geometry directly.

Instead it stores a **reference to the wall face**.

Conceptually:

```
SketchEdge
{
   type: HOST_REFERENCE
   hostElementId: Wall_42
   hostReference: ExteriorFace
   offset: 0
}
```

So the slab boundary edge becomes:

```
Boundary = WallFace + Offset
```

Not:

```
Boundary = Static Line
```

---

# 3️⃣ The Key System: Reference Geometry

Revit elements expose **reference planes and faces**.

Walls expose references like:

```
Wall
 ├─ CenterLine
 ├─ CoreExterior
 ├─ CoreInterior
 ├─ FinishExterior
 └─ FinishInterior
```

When a slab edge references a wall:

```
SlabEdge → Wall.ReferencePlane
```

So the dependency graph becomes:

```
Wall Geometry
      ↓
Wall Reference
      ↓
Slab Sketch Edge
      ↓
Slab Solid
```

---

# 4️⃣ What Happens When the Wall Moves

When you move a wall:

1️⃣ Wall geometry updates
2️⃣ Wall references move
3️⃣ Slab sketch edge recomputes
4️⃣ Slab profile regenerates
5️⃣ Slab solid rebuilds

So the slab edge **follows the wall automatically**.

Dependency chain:

```
Move Wall
   ↓
Update Wall Face
   ↓
Update SketchEdgeReference
   ↓
Recompute Slab Profile
   ↓
Rebuild Slab Geometry
```

---

# 5️⃣ Offsets When Picking Walls

Revit allows offsets when picking walls.

Example:

```
SlabEdge = WallExteriorFace + 200mm
```

Stored as:

```
SketchEdge
{
  hostId: Wall_42
  reference: ExteriorFace
  offset: 0.2
}
```

If wall thickness changes:

```
ExteriorFace moves
     ↓
Edge recomputes
```

---

# 6️⃣ What Happens if the Wall Is Deleted

Revit must preserve the slab.

So the reference becomes **unconstrained geometry**.

Revit converts:

```
HostReferenceEdge
```

into

```
FreeEdge
```

This prevents slab collapse.

---

# 7️⃣ How Revit Handles Complex Shapes

A slab boundary is **a set of closed loops**:

```
Sketch
  loops[]
     edges[]
```

Example:

```
OuterLoop
   edge1
   edge2
   edge3
   edge4

InnerLoop (opening)
   edge5
   edge6
   edge7
   edge8
```

Each edge can be:

```
line
arc
spline
host reference
```

---

# 8️⃣ Revit’s Constraint Solver

Revit runs a **parametric constraint solver**.

Every edge has constraints:

```
Edge A must lie on Wall_42 exterior face
Edge B must remain perpendicular to Edge A
Edge C must stay horizontal
```

The solver ensures:

```
Valid closed profile
No intersections
Consistent geometry
```

If constraints conflict:

Revit shows **“Constraints are not satisfied”**.

---

# 9️⃣ How Geometry Is Rebuilt

After solving constraints:

1️⃣ Profile is recomputed
2️⃣ Slab extrusion is generated

```
Solid = Extrude(profile, thickness)
```

This is regenerated by the **Revit geometric kernel**.

---

# 🔟 Why This Is Powerful

This system gives Revit:

### Associativity

Slabs follow walls automatically.

### Parametric behavior

Change a wall → slab updates.

### Non-destructive modeling

Sketches stay editable.

# 1️⃣1️⃣ What This Means for Your BIM Engine

To replicate Revit behavior, you need:

### 1️⃣ Sketch profiles

```
Slab
  sketch
     loops[]
        edges[]
```

---

### 2️⃣ Edge references

```
Edge
{
  type: "host"
  hostId: wallId
  reference: "exteriorFace"
  offset: number
}
```

---

### 3️⃣ Dependency graph

```
Wall
  ↓
SketchEdgeReference
  ↓
Slab
```

---

### 4️⃣ Rebuild propagation

When wall changes:

```
WallStore.update
   ↓
DependencyResolver
   ↓
SlabBuilder.rebuild
```

---

# 1️⃣2️⃣ The Core Idea

Revit **does not store geometry**.

It stores **rules that generate geometry**.

```
Geometry = Function(Constraints, Parameters)
```

Example:

```
SlabProfile =
   Wall42.ExteriorFace
   + Wall43.ExteriorFace
   + FreeEdge
```

---

# 1️⃣3️⃣ The Hard Part (Why BIM Engines Struggle)

The difficult parts are:

### Constraint solving

Detecting valid profiles.

### Reference tracking

Keeping host relationships.

### Regeneration order

Updating elements deterministically.

---

# 💡 Key Concept

Revit is fundamentally a **parametric constraint system**, not a geometry editor.

Geometry is just the **result of solving relationships between elements**.
