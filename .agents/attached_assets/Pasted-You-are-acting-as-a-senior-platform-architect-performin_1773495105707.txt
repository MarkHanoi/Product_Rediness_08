You are acting as a senior platform architect performing a full architectural audit of this codebase.

Your objective is to evaluate whether the platform layer architecture is correctly implemented according to the following contract:

BIM PLATFORM UI CONTRACT
(Platform Layer — Mandatory for All Platform UI Code Modifications)

The platform entry flow is:

LandingPage → AuthModal → ProjectHub → BIM Workspace

PlatformRouter.start(engineInit) is the only legal entry point.

The platform layer must:
- run before the BIM engine initializes
- be completely isolated from BIM engine modules
- use pure DOM manipulation (no frameworks)
- mount only inside #platform-root
- never touch the Three.js canvas (#container)

The platform components are:

PlatformRouter
LandingPage
AuthModal
ProjectHub
PlatformShell

Each component:
- must be a TypeScript class
- must implement destroy()
- must build DOM in build()/render() methods
- must follow CSS prefix ownership

The BIM engine is initialized ONLY after a project is opened.

The engine initialization flow is:

User selects project
→ PlatformRouter.launchWorkspace()
→ 420ms fade out
→ engineInit()
→ shell.setProjectContext()

Projects and users are currently stored in localStorage.

localStorage keys:
- bim-platform-user
- bim-projects-index

The architecture intentionally defers engine initialization.

Your task is to analyze the codebase and produce a **complete architecture audit report**.
