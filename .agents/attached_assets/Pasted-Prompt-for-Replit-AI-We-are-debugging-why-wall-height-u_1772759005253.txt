We are debugging why wall height updates do not visually update the wall after changing properties in the Properties Inspector.

Current behavior:

window.wallStore is now correctly defined.

Wall properties appear when selecting a wall.

Changing a property and clicking Save Changes:

The wall does not update visually

The inspector loses the properties afterward

We suspect the issue is in the Command → Store → Builder pipeline.

Please perform the following diagnostic checks and fixes carefully without breaking the architecture.

1️⃣ Verify WallStore.update() returns the updated wall

Open:

src/stores/WallStore.ts

Locate:

update(id, patch)

Ensure the function returns the updated wall object, not a boolean.

Correct implementation:

update(id: string, patch: Partial<WallData>): WallData | undefined {
    const wall = this.walls.get(id);
    if (!wall) return;

    const updated = {
        ...wall,
        ...patch
    };

    this.walls.set(id, updated);

    return updated;
}

If the function currently returns true or boolean, fix it.

Commands rely on receiving the updated wall object.

2️⃣ Verify UpdateWallHeightCommand is using CommandContext

Open:

src/commands/walls/UpdateWallHeightCommand.ts

Find this code:

const builder = (window as any).wallFragmentBuilder || (window as any).wallBuilder;

    builder.updateWall(updated);
}

Replace it with:

const builder = ctx.wallFragmentBuilder;

if (builder?.updateWall) {
    builder.updateWall(updated);
}

Commands must not depend on window globals.

export interface CommandContext {
