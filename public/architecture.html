— Key Data Flows (Sequence Diagrams)

🖊️ User Edits an Element (e.g. moves a wall)

1
UI — pointer event on canvas → SelectionManager
2
Tool — WallTransformController → drag delta
3
SnapManager — snap to grid / geometry
4
CommandManager → execute(UpdateWallCommand) + Immer snapshot
5
WallStore mutated → StoreEventBus fires
6
Renderer rebuilds Three.js mesh for wall
7
SyncStateEngine → diff → POST delta to server
8
Socket.io broadcasts command-executed to other clients

🤖 AI Floor Plan Generation

1
BriefInputPanel — user types brief text
2
POST /api/ai/brief/parse — Claude parses to GenerativeDesignBrief JSON
3
LayoutGenerator — constraint-based room placement
4
GenerativeDesignAdvisor — validates constraints; if fail → /api/ai/generative/advise
5
FloorPlanBatchExecutor → FloorPlanCommandBatcher builds command batch
6
CommandManager → executes each CreateWall / CreateRoom command
7
VariantBrowserPanel shows generated variants

💾 Save / Open Project

1
ProjectsRailPanel → "Save" action
2
ProjectSerializer — serialises all stores → JSON snapshot
3
POST /api/projects/:id/save with auth JWT
4
pgProjectStore — upserts to Replit PostgreSQL (or Supabase)
5
versionStateMachine — records ISO 19650 state transition
6
Open: ProjectLoader → MigrationEngine upgrades schema if needed → initStores

🖥️ App Startup (cold load)

1
index.html → main.ts loads Platform UI (no engine bundle)
2
Landing page + auth — login/signup with authStore JWT
3
User opens project → dynamic import("EngineBootstrap")
4
initStores → registerAllStores
5
createRenderer → WebGPU (or WebGL2 fallback)
6
initBuilders → initTools → initUI (parallel with WASM init)
7
fragments.init() WASM — non-blocking, deferred until IFC export needed
8
initPersistence → SyncStateEngine → Socket.io handshake
